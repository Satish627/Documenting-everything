# Spikes

Short research/design docs written *before* committing to a build — Franklin's rule of thumb:
**≈5 minutes to read, ≤30 minutes of research.** A spike de-risks an unclear task and gives
your manager something concrete to align on.

Since there's no system access yet, spikes here are **design + research only**, built on the
assumed stack (Next.js + TypeScript on Vercel, PostgreSQL). Every assumption is marked to confirm on day 1.
